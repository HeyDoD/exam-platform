# 經濟部新進職員甄試「資訊」類別：專業科目 B 綜合學習指南

本學習指南係針對經濟部所屬事業機構 110 年至 114 年新進職員甄試「專業科目 B（資訊管理、程式設計）」之歷屆試題進行深度解析與綜整。

---

## B-01 資訊系統分析與設計

### 關鍵概念定義
*   **SSDLC (Secure System Development Life Cycle)**：依據國家資通安全防護整合服務計畫，指在系統開發生命週期中納入安全控制措施，確保 WEB 應用程式之安全性。
*   **UML (Unified Modeling Language)**：統一建模語言，用於系統設計與分析之標準化語言，協助分析師與設計師視覺化、設計並記錄系統構造。

### 常考申論方向
1.  **SSDLC 各階段控制措施 (111年)**：要求說明開發、測試、部署與維運階段之具體安全控制項目。
2.  **UML 圖表應用 (113年)**：列舉並繪製說明 4 種常用 UML 圖表。

### 答題架構建議
*   **SSDLC 類：** 建議採表格或條列式，依「開發（5項）」、「測試（2項）」、「部署維運（3項）」層次分明地列出控制措施。
*   **UML 類：** 先定義 UML 目的，再分點說明圖表名稱（如：使用案例圖、類別圖等），並搭配簡圖與適用場景描述。

## B-02 資訊管理概論

### 關鍵概念定義
*   **數位轉型 (Digital Transformation)**：企業利用數位科技進行全面性變革，以在後疫情時代下存續。
*   **物聯網 (IoT)**：涵蓋四項主要技術範疇之聯網架構。
*   **機器學習 (Machine Learning)**：包含監督式學習與非監督式學習兩大實務應用類別。

### 常考申論方向
1.  **數位轉型之推動 (112年)**：定義數位轉型、成功轉型須兼顧的 5 大層面，以及轉型經歷的 3 個階段。
2.  **機器學習分類 (111年)**：比較監督式與非監督式學習之定義、處理問題及演算法。
3.  **物聯網與資訊倫理 (110年)**：說明 IoT 範疇與資訊倫理之定義。

### 答題架構建議

## B-03 系統程式

### 關鍵概念定義
*   **Stack (堆疊)**：一種「後進先出」(LIFO) 的資料結構。
*   **Garbage Collection (垃圾回收)**：Java 等語言中自動管理記憶體之機制，負責回收不再使用的物件空間。

### 常考申論方向
1.  **Stack 的實務應用與實作 (112年)**：定義 Stack、列舉 2 個應用案例（如回溯、函式呼叫），並撰寫基本操作程式碼。
2.  **Java 記憶體管理 (114年)**：說明垃圾回收之運作原理及觸發時機。
3.  **DBMS 效能調校 (113年)**：計算 B+ tree 索引架構下，不同 SQL 語法存取硬碟頁 (Disk Page) 的次數。

### 答題架構建議
*   **計算類：** 必須列出詳細計算過程，例如：`總筆數 / 每頁筆數 = 葉節點頁數`，加上索引層級數目。
*   **原理類：** 清楚說明「何時觸發」（如：記憶體不足時）與「如何運作」。

## B-04 資料庫管理

### 關鍵概念定義
*   **Relational Database (關聯式資料庫)**：使用資料表 (Table) 儲存資料，並透過主鍵 (Primary Key) 與外鍵 (Foreign Key) 建立關聯。
*   **SQL 萬用字元與 EXISTS**：進階查詢語法，用於模糊比對及子查詢存在性判斷。

### 常考申論方向
1.  **複雜 SQL 查詢 (110-113年)**：包含排序 (ORDER BY)、聚合計算 (SUM, AVG)、分組 (GROUP BY) 及多表關聯 (JOIN)。
2.  **DDL 與索引建置 (113年)**：建立資料表、定義主外鍵約束及建立 CLUSTER INDEX。
3.  **進階篩選 (111-112年)**：使用萬用字元篩選特定模式之字串，或四捨五入計算銷售折扣。

### 答題架構建議
*   **SQL 指令撰寫：**
    1.  注意關鍵字大小寫（慣例上）。
    2.  明確標註 Table JOIN 的條件。
    3.  若題目要求計算，需在 SELECT 項目中使用運算式。

## B-05 網路管理

### 關鍵概念定義
*   **TTL (Time To Live)**：封包生存時間，防止封包在網路中無限循環，也是 Traceroute 之核心原理。
*   **非對稱路由 (Asymmetric Routing)**：封包去程與回程經過不同路徑之現象。
*   **DNSSEC/DoH/DoT**：強化域名解析安全之技術，DoH/DoT 著重加密傳輸，DNSSEC 著重資料完整性驗證。

### 常考申論方向
1.  **網路診斷與 Traceroute (114年)**：解釋 TTL/ICMP 角色、非對稱路徑成因，以及排查「偶發高延遲」之測試矩陣。
2.  **IPv4 與 IPv6 比較 (114年)**：從位址長度、標頭大小、分片、QoS 等 10 種面向進行對比。
3.  **DNS 安全 (114年)**：說明 DoH/DoT 與 DNSSEC 如何在功能上互補。

### 答題架構建議
*   **對比類：** 強烈建議使用 10 列之表格，逐項對齊 IPv4 與 IPv6 之異同。
*   **排查類：** 依題目給予的維度（協定、封包大小、路徑等）分項說明診斷方法。

## B-06 電子商務

### 關鍵概念定義
*   **雲端運算服務模型**：
    *   **IaaS (基礎設施即服務)**：提供基礎運算資源。
    *   **PaaS (平台即服務)**：提供開發平台。
    *   **SaaS (軟體即服務)**：透過網路直接使用軟體。

### 常考申論方向
1.  **雲端服務分類與實例 (110, 113年)**：定義 3 種服務種類，並提供 CSP (雲端服務提供者) 實例（如 Google Gmail 屬 SaaS）。
2.  **雲端部署模型 (112年)**：說明 4 種部署模型（公有雲、私有雲、混合雲、社群雲）之特性。

### 答題架構建議
*   **定義類：** 先給出標準定義，再列舉具體業者或產品名稱作為佐證。

## B-07 資訊安全管理

### 關鍵概念定義
*   **ISO 27001 (ISMS)**：資訊安全管理系統國際標準。
*   **PDCA 循環**：Plan-Do-Check-Act，持續改進之架構。
*   **Web 攻擊行為**：
    *   **SQL Injection**：惡意注入 SQL 指令。
    *   **XSS (Cross-Site Scripting)**：跨站腳本攻擊。
    *   **CSRF (Cross-Site Request Forgery)**：跨站請求偽造。

### 常考申論方向
1.  **資安技術防護措施 (110年)**：針對硬體及軟體之 5 個資安防護技術。
2.  **ISMS 維運 (113年)**：繪圖說明 ISO 27001 之 PDCA 循環。
3.  **雲端資安指引 (112年)**：機關使用雲端服務應考量之 6 個資安政策項目。
4.  **網站攻擊與防禦 (110, 114年)**：說明攻擊原理及防範對策。

### 答題架構建議
*   **防禦類：** 採取「攻擊原理」＋「防禦措施」兩段式回答。
*   **管理類：** PDCA 需畫出圓形循環圖，並分別標註各階段（P-規劃、D-實施、C-監控、A-改進）之內涵。

## B-08 程式設計

### 關鍵概念定義
*   **Recursion (遞迴)**：函式呼叫自身以解決問題。
*   **Sorting (排序)**：Quick Sort (快速排序)、Insertion Sort (插入排序) 等演算法。

### 常考申論方向
1.  **遞迴演算 (110-111年)**：費氏數列 (Fibonacci)、最大公因數 (GCD)、細胞分裂總數計算。
2.  **排序實作 (111, 113年)**：撰寫 Quick Sort 或 Insertion Sort 程式碼，並考慮遞增/遞減參數。
3.  **演算法邏輯 (112, 114年)**：資料清理腳本（驗證 ID、手機、Email）、快樂數 (Happy Number) 判斷。

### 答題架構建議
*   **程式碼撰寫：**
    1.  注意遞迴之「停止條件」。
    2.  若題目要求「非遞迴」方式，應使用迴圈實作。
    3.  需注意邊界條件驗證（如手機 10 位數、09 開頭）。

## B-09 物件導向分析與設計

### 關鍵概念定義
*   **封裝 (Encapsulation)**：隱藏物件細節，僅暴露介面。
*   **繼承 (Inheritance)**：子類別繼承父類別之特徵與行為。
*   **多型 (Polymorphism)**：同一介面可有多種實現方式。

### 常考申論方向
1.  **Java OOP 三大特性 (114年)**：說明封裝、繼承、多型之定義及在 Java 中如何實現（如 `private`, `extends`, `interface`）。
2.  **型別比較 (114年)**：比較基本型別 (Primitive) 與參考型別 (Reference) 之差異。

### 答題架構建議
*   **特性說明：** 應配合 Java 語法關鍵字進行解釋，使回答更具體。

## B-10 ERP 系統與商業應用

### 關鍵概念定義
*   **業務邏輯實作**：將企業營運規則（如會員折扣、銷售統計）轉化為資料庫查詢。

### 常考申論方向
1.  **銷售與折扣計算 (110, 112年)**：計算會員 vs 非會員之購買總額，或因輸入錯誤導致的損失（92 折誤植為 82 折）。
2.  **人事資料管理 (111年)**：撰寫程式檢查員工代號校驗碼，並篩選特定部門之特別費合計。
3.  **排程與隊列應用 (110年)**：設計取號機服務 (GetNumber) 及電梯調度系統 (SelElev)。

### 答題架構建議
*   **商業邏輯類：** 需仔細閱讀題目給予的折扣公式或校驗規則（如：第 6 碼為前 5 碼總和之個位數），並將其精確轉化為 SQL 或程式判斷式。

### 關鍵詞彙表 (Glossary)
*   **Primary Key (主鍵)**：資料表中唯一識別每一筆記錄的欄位。
*   **B+ Tree**：常見的資料庫索引結構，影響硬碟存取效率。
*   **Traceroute**：利用 ICMP 與 TTL 遞增來追蹤封包路徑的工具。
*   **Asymmetric Routing**：封包往返路徑不一致，可能導致網路診斷困難。
*   **AVL Tree**：一種自我平衡二元搜尋樹。
*   **Insertion Sort (插入排序)**：將未排序資料逐一插入已排序數列之適當位置。
